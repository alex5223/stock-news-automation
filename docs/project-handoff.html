1. 專案目標

建立一個每日自動化的股票投資研究輔助系統。系統每天收集 YouTube 財經影片與財經新聞 RSS，整理當日重要消息、重複被提到的產業與個股，並輸出可追蹤的每日報告與歷史資料。

定位是「研究輔助」而非自動下單或個人化投資顧問。頻率與聲量代表討論熱度，不等於基本面改善或股價方向。

5. 目前設定重點

  • 排程時區：Asia/Taipei
  • 資料視窗：rolling 24 hours
  • lookback days：5
  • YouTube：預設啟用，使用 YOUTUBE_API_KEY
  • RSS：預設啟用，但 feed URL 目前留空，需要填入官方 RSS/Atom URL
  • LLM：預設關閉，模型預設 gpt-5.5
  • local storage：預設啟用，寫入 data/runtime
  • history snapshot：預設啟用，寫入 data/history
  • Google Sheets：預設關閉

需要的環境變數

  • YOUTUBE_API_KEY：YouTube Data API v3 key
  • OPENAI_API_KEY：開啟 LLM 摘要時需要
  • OPENAI_MODEL：可覆蓋預設模型
  • GOOGLE_APPLICATION_CREDENTIALS：本機 Google service account JSON 路徑
  • GOOGLE_SHEET_ID：Google Sheet ID
  • GOOGLE_SERVICE_ACCOUNT_JSON：GitHub Actions secret，用於建立暫存 service account 憑證檔

6. GitHub Actions 狀態

目前 workflow 路徑：.github/workflows/daily-report.yml

  1. 使用 workflow_dispatch 與每日排程。
  2. 權限已改為 contents: write，讓 GITHUB_TOKEN 可以 push commit。
  3. 安裝 Python 3.11 與專案依賴。
  4. restore data/runtime cache。
  5. 準備 Google service account credentials。
  6. 執行 python -m stock_news_bot --config config/sources.yaml。
  7. commit/push data/history/。
  8. save runtime cache，並上傳 Markdown 報告 artifact。
git config user.name "github-actions"
git config user.email "actions@github.com"
git add data/history/
if git diff --cached --quiet; then
  echo "no history snapshot changes"
else
  git commit -m "chore: daily snapshot $(TZ=Asia/Taipei date +%F)"
  git pull --rebase
  git push
fi

7. 已知限制與需注意事項

測試尚未在目前環境跑過。

嘗試執行 python -m pytest 時，sandbox 回傳 python.exe 無法執行。這表示目前沒有完成實際測試驗證，下一個環境應先安裝/確認 Python 後跑測試。

  • RSS URL 尚未填入，實際新聞收集需先確認各站官方 feed。
  • YouTube 需要有效 API key；字幕抓取可能因影片、語言或 YouTube 限制而失敗。
  • OpenAI LLM 摘要預設關閉，打開前需確認模型名稱與帳號權限。
  • data/history 會保存來源文字與報告全文，長期執行會讓 repo 變大。
  • 目前 pipeline 合併歷史資料時會讀 data/runtime/daily_entities.csv 與 data/history/*.json；若希望 committed snapshots 成為唯一正式來源，可以調整 priority 或移除 runtime history 依賴。
  • 檔案清單中出現 src/stock_news_sentinel.egg-info/ 與 docs/print(1).py，需確認是否為不小心產生的暫存/測試檔。尚未刪除，避免誤刪使用者資料。
